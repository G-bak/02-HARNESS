# TASK-20260429-009 보고서 — Generator 파이프라인 실제 실행 + 2개 wrapper 버그 발견·기록

## 핵심 요약

- **상태:** 완료. Generator의 실제 작업(`docs/guides/validator-pipeline-ko.md`에 secret-scan 운영 노트 1줄 추가)은 7번째 시도에서 실제 Claude Code CLI 호출로 정상 수행됐다.
- **결론:** 6번 실패 + 1번 성공 과정에서 **wrapper의 두 가지 버그를 격리·기록**했다. (1) `--bare` 플래그가 OAuth 자동 로드를 silently skip — wrapper에서 제거로 즉시 해결. (2) `--json-schema`는 응답 텍스트 내용을 강제하지 않아 wrapper의 stdout JSON 파싱이 자연어 응답에 실패 — TASK-010으로 분리.
- **서비스 영향:** 제품 런타임 영향 없음. 운영 가이드 1건(목표 산출물) + 운영 가이드 2건(gotcha 소급 적용) + wrapper 1건(--bare 제거) + 인사이트 4건 + TASK-010 spec 1건.
- **검증:** Generator의 file edit는 git diff로 직접 확인됨. `npm run audit:harness`, `npm run check:encoding` PASS. Validator-A 외부 Codex CLI 실행은 별도 세션으로 사용자가 이어서 진행 예정.
- **남은 리스크:** wrapper output parsing은 여전히 자연어 응답에 실패함 — TASK-010이 해결할 때까지 운영자는 git diff로 수동 검증 필요. `--bare` 제거로 hooks·plugins·CLAUDE.md auto-load 활성 — 컨텍스트 격리는 권한 모드와 도구 allowlist에 의존하게 됨.
- **조치 필요:** 사용자가 별도 세션에서 Codex 측 Validator-A 실제 실행 검증 진행. 그 후 TASK-010 (wrapper output parsing fix) 진입.

## 6실패 + 1성공 — 시도별 진단

| # | 환경 | 실패 모드 | exit | 원인 |
|---|---|---|:---:|---|
| 1 | Codex sandbox → `claude` (PATH) | `spawnSync ENOENT` | - | binary not in PATH |
| 2 | Codex sandbox → `claude.cmd` | `spawnSync EINVAL` | - | Windows .cmd 호출 실패 |
| 3 | Codex sandbox → `claude.exe` | `Not logged in · /login` | 1 | Codex sandbox가 OAuth 안 넘김 |
| 4 | Claude Code session → `claude.exe` | `Not logged in · /login` | 1 | `CLAUDECODE=1` 환경에서 자식이 SSE port로 부모 의존 → 실패 |
| 5 | Claude Code → PowerShell tool (env 정리) → `claude.exe` | `Not logged in · /login` | 1 | env 정리해도 동일 → 환경 변수 문제 아님 확인 |
| 6 | + `CLAUDE_CODE_OAUTH_TOKEN` env var | `Not logged in · /login` | 1 | 토큰 있어도 fail → 토큰 문제 아님 확인 |
| 7 | + `--bare` 제거 | **PASS** | 0 | `--bare`가 OAuth 자동 로드를 skip하는 게 진짜 원인 확정 |

진단 결정타: 7번째 시도 직전 A/B 테스트 — `claude --print` (no --bare): 성공 ($0.16). 같은 세션 같은 토큰으로 `claude --bare --print`: 실패. **단일 변수 `--bare`로 격리됨.**
