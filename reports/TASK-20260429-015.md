# TASK-20260429-015 보고서 — Generator → Validator-A 실제 smoke test

## 핵심 요약

- **상태:** PARTIAL. Generator 실제 실행은 성공했고, Validator-A 실제 실행은 CLI Resource Failure로 완료하지 못했다.
- **결론:** `run-generator.mjs`는 실제 Claude Code CLI로 smoke target을 생성했다. `run-validator-a.mjs`는 현재 Windows 환경에서 Codex CLI shim을 `spawnSync`로 실행하지 못한다.
- **서비스 영향:** 제품 런타임 영향 없음. task-scoped smoke artifact만 생성.
- **검증:** Generator 결과 JSON, smoke target 파일, Generator run metadata는 생성됐다. Validator-A는 `codex ENOENT`, `codex.cmd EINVAL`, extensionless shim `ENOENT`로 모두 Resource Failure.
- **남은 리스크:** Validator-A wrapper는 Windows npm shim 실행 보강 없이는 이 환경에서 실제 Codex 모델 검증까지 도달하지 못한다.
- **조치 필요:** 후속 Task에서 `run-validator-a.mjs`가 Windows `.cmd`/PowerShell/npm shim을 안전하게 실행하도록 보강한 뒤 같은 smoke를 재시도한다.

## 발견 사항

1. `run-generator.mjs`는 현재 `main...HEAD` 전체 branch diff를 검사하므로, 같은 Task에서 준비 파일을 먼저 커밋한 smoke/실사용 흐름에서는 setup 파일이 out-of-scope로 보일 수 있다. 더 정확한 방식은 실행 전 baseline을 잡고 subprocess가 새로 만든 변경만 검증하는 것이다.
2. `run-validator-a.mjs`는 Windows npm shim을 직접 `spawnSync`할 때 실패한다. `codex.cmd`는 `EINVAL`, 기본 이름은 현재 Node subprocess PATH에서 `ENOENT`가 난다. Windows에서는 `cmd.exe /d /s /c` 경유 또는 실제 JS entrypoint를 명시 실행하는 보강이 필요하다.
