## 1. 요청 및 목표

사용자는 Claude CLI가 제시한 review 의견을 바탕으로 Generator wrapper를 실제 운영 전에 보강하라고 지시했다.

목표는 다음 리스크를 제거하는 것이다.

- 문서화되지 않은 `best` 모델 alias 기본값
- git write 권한의 기본 허용 여부 모호성
- system guidance와 handoff JSON이 stdin에 섞이는 구조
- `refs.spec`, `refs.ledger` 실존 검증 부족
- 동일 Task 동시 실행 충돌 가능성
- stderr artifact의 민감 정보 취급 주의 부족

## 2. 결과 및 변경 사항

- `scripts/run-generator.mjs`
  - 기본 모델을 `best`에서 공식 alias인 `opus`로 변경
  - `--append-system-prompt`로 system guidance 분리, stdin은 handoff payload만 전달
  - `refs.spec` 파일 존재와 `refs.ledger` task 원장 경로 검증 추가
  - 실제 실행 시 `.generator.lock` 생성/해제로 동일 Task 동시 실행 차단
  - git write 권한은 기본 차단, `--allow-git-write` 명시 시에만 추가
  - `maxBuffer`를 20MB에서 50MB로 상향
- `docs/guides/generator-pipeline-ko.md`
  - `opus` 기본값, git write opt-in, lockfile, ref 검증, stderr 주의 문서화
- `docs/guides/researcher-procedure-ko.md`
  - Researcher 외부 CLI fallback 모델 선택 기준 보강
- `CURRENT_STATE.md`, `logs/sessions/SESSION-20260428-001.md`, `logs/tasks/TASK-20260428-002.jsonl`
  - 활성 Task와 검증 기록 반영

## 4. 영향도 및 리스크

제품 코드, API, UI, DB, 배포 설정은 변경하지 않았다.

운영 영향:

- 실제 Claude CLI 호출 실패 가능성이 큰 `best` alias 기본값을 제거했다.
- Generator가 실수로 git write까지 수행하는 기본 위험을 줄였다.
- handoff payload가 system 지침과 섞이지 않게 분리했다.
- 잘못된 Task Spec이나 원장 경로로 실행되는 것을 사전에 막는다.
- 같은 Task를 동시에 실행해 artifact와 원장이 충돌하는 상황을 방지한다.

남은 리스크:

- 실제 Claude CLI 생성 실행은 계정/권한/모델 지원 상태에 따라 별도 검증이 필요하다.
- Validator-A 실행 wrapper와 retry handoff 자동 생성은 아직 후속 Task다.
