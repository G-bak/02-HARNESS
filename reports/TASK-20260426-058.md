## 2. 변경 사항

- `docs/operations/git-branch-policy.md`
  - PowerShell에서는 Git 명령을 `&&`로 묶지 않고 한 단계씩 실행하도록 명시했다.
  - squash merge 운영 순서를 추가했다.
  - self-referential squash commit hash는 원장 내부에 고정 해시를 반복 amend하지 않고 `SELF_REFERENTIAL_MAIN_SQUASH_COMMIT` 표기를 허용하도록 정리했다.
  - push를 수행하지 않은 경우 `main`이 `origin/main`보다 앞선 상태를 최종 보고와 `CURRENT_STATE.md`에 남기도록 했다.
- `docs/operations/work-history-policy.md`
  - Git 상태 인수인계 규칙을 추가했다.
  - 현재 브랜치, clean/dirty, ahead/behind, task branch commit, main merge, push 여부를 필수 기록 항목으로 명시했다.
- `docs/workflows/task-lifecycle.md`
  - 최종 보고와 완료 조건에 git 상태, commit, main/origin ahead, push 여부를 포함했다.
- `CURRENT_STATE.md`
  - 현재 로컬 `main`이 `origin/main`보다 앞서 있고 push는 아직 수행하지 않았다는 인수인계 문구를 추가했다.
- `scripts/check-completion-gates.mjs`
  - 활성 Task가 있는 동안에는 현재 작업으로 인한 dirty worktree를 직전 완료 Task의 누락으로 오인하지 않도록 조건을 좁혔다.
