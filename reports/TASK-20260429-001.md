# TASK-20260429-001 보고서 — 2-commit squash + 인사이트 분류 자동 강제

## 핵심 요약

- **상태:** 작업 완료. main 2-commit squash 머지 + push + 자기 브랜치 자동 정리 회귀 테스트 직전.
- **결론:** TASK-20260428-006에서 발견한 두 가지 문제(metadata로 인한 자기 브랜치 자동 정리 실패 + 인사이트가 로그에만 남아 다음 세션에서 잊힘)를 정책·자동 감사·소급 적용으로 동시에 해결했다.
- **서비스 영향:** 제품 런타임 영향 없음. 권위 문서 2건, 자동 감사 스크립트 1건, 인사이트 로그 보강.
- **검증:** `npm run audit:harness`, `npm run check:encoding` 모두 PASS. 새 게이트는 본 Task의 amendment 인사이트(category=gotcha + applied=applied)를 정상 통과시키고, observation 인사이트는 게이트 대상에서 자동 제외됨을 확인.
- **남은 리스크:** 새 게이트는 `applied_to_doc.commit` 필드 존재만 확인 — 실제로 그 commit에 가이드 변경이 들어갔는지까지는 검증 안 함. 향후 강화 후보.
- **조치 필요:** 사용자 승인은 이미 받음. 즉시 commit → 2-commit squash → push → 자기 브랜치 정리 진행.

## 사용자 요구

> "그냥 로그로 남기는 건 까먹기 쉬워 그래서 아예 그냥 가이드 문서 자체를 수정하게끔 하는건 어때?"

→ 옵션 C 채택: 인사이트 분류 + 자동 차단 + INS-006-03 소급 적용. 동시에 직전 발견된 squash policy edge case도 옵션 (a) 2-commit 패턴으로 해결.
