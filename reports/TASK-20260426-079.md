# Executive Summary

LandingHub와 정확히 같은 형태로 `랜딩페이지 쇼케이스 + 뉴스레터 운영`을 동시에 깊게 제공하는 유명 서비스는 많지 않다. 시장은 실제로 `인스피레이션/갤러리 계열`과 `뉴스레터/오디언스 플랫폼 계열`로 나뉘어 있다.
따라서 LandingHub의 경쟁 분석은 두 축을 함께 보는 편이 정확하다.

핵심 결론은 다음과 같다.

1. 쇼케이스 축에서는 `탐색 깊이`와 `재사용성`이 부족하다.
2. 뉴스레터 축에서는 `세분화/자동화`, `실험`, `성장 루프`가 가장 큰 갭이다.
3. 단순 발송 도구 수준을 넘어서려면 `구독자 데이터 활용`과 `쇼케이스 발견 경험`을 동시에 강화해야 한다.

## LandingHub Missing Features

### H

| 기능 갭 | 이유 |
|---|---|
| 고급 쇼케이스 탐색: 산업/스타일/플랫폼/색상/디바이스/페이지 타입 필터 | 유명 갤러리 대부분의 기본값이다. 현재 카드 나열만으로는 탐색 효율이 낮다. |
| 섹션/컴포넌트 단위 탐색 | 실무자는 전체 페이지보다 `Hero`, `Pricing`, `FAQ` 같은 단위 검색을 기대한다. |
| 저장/보드/북마크 | 반복 방문 사용자는 큐레이션 소비보다 수집 기능을 기대한다. |
| 구독자 세그먼트/태그/자동화 | 뉴스레터 플랫폼 최소 기대치다. 소스별 태깅, 웰컴 시퀀스, 휴면자 분기 등이 필요하다. |
| A/B 테스트와 성과 최적화 | 제목/본문/발송 시간 테스트가 없으면 운영 퀄리티가 금방 한계에 부딪힌다. |
| 성장 루프: 추천/리퍼럴/크로스프로모션 | beehiiv, Kit, Substack는 모두 네트워크 성장 장치를 갖고 있다. LandingHub는 리스트 성장 장치가 약하다. |
| 전환 분석: 구독 폼/캠페인/소스별 성과 | 단순 발송 이력만으로는 운영자가 무엇이 먹히는지 알기 어렵다. |

## Recommended Product Direction

LandingHub가 다음 단계로 가려면 아래 순서가 합리적이다.

1. `탐색 강화`: 필터, 태그, 섹션 단위 탐색, 북마크
2. `오디언스 운영 강화`: 세그먼트, 태그, 자동화, 더블 옵트인
3. `최적화 강화`: A/B 테스트, 전환 분석, 소스 추적
4. `성장 강화`: 추천/리퍼럴/크로스프로모션
5. `확장`: 템플릿 자산, 커뮤니티, 수익화
