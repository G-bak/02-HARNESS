# Context Management — 컨텍스트 관리 전략

**버전:** 1.0 | **최종 수정:** 2026-04-24  
**원칙:** 길이보다 high-signal. 목표 달성에 직접 필요한 정보만 남겨야 모델이 집중할 수 있다.

---

## 컨텍스트 최소화 원칙

**전체 공유 금지.** 각 에이전트에게는 해당 작업 수행에 필요한 최소 컨텍스트만 전달한다.

과도한 컨텍스트가 유발하는 문제:
- 에이전트의 판단 오염
- 토큰 낭비 및 응답 품질 저하
- 민감 정보 노출 위험

---

## 계층적 컨텍스트 관리 (L1 / L2 / L3)

| 계층 | 대상 | 보존 수준 | 압축 기준 |
|---|---|---|---|
| **L1 활성** | 현재 진행 중인 Task | 전체 보존 | 압축 없음 |
| **L2 완료** | 동일 세션 내 완료된 Task | 구조화 요약 | 실행 로그 압축, 핵심 정보 보존 |
| **L3 아카이브** | 이전 세션의 Task | 핵심 참조 | task_id + 결과 + 교훈만 보존 |

---

## L2 압축 시 보존 필수 항목

완료된 Task를 L2로 압축할 때 **반드시 보존**:

```
✅ task_id 및 완료 시각
✅ 요청 목표 요약 (원문 압축, 의미 보존)
✅ 최종 결과 및 결과물 위치 (파일 경로, 커밋 해시)
✅ 주요 의사결정과 그 근거
✅ 발생한 오류와 해결 방식
✅ 가이드 문서에 반영할 인사이트 ([TAG] 포함)
✅ 부정적 제약 조건 (Negative Constraints) — 아래 참조
```

## L2 압축 가능 항목

```
❌ (압축 가능) 에이전트 간 중간 통신 로그
❌ (압축 가능) 재시도 과정의 중간 실패 상세
❌ (압축 가능) 탐색 과정의 원문 데이터
❌ (압축 가능) 검증 단계의 개별 체크 결과 (최종 판정만 보존)
```

---

## 절대 압축 금지 항목

어떤 계층에서도 삭제하거나 단순 요약하지 않는다:

```
🔒 실패의 근본 원인 분석 (같은 실패 반복 방지)
🔒 가이드 규칙을 변경하게 된 경험적 근거
🔒 보안 관련 판단 및 민감 정보 처리 결정
🔒 사용자와 합의한 요구사항 변경 이력
🔒 부정적 제약 조건 (Negative Constraints)
```

---

## 부정적 제약 조건 (Negative Constraints) 특별 처리

**"이것은 하지 마세요"** 라는 정보는 요약 과정에서 가장 먼저 유실되는 정보다.

- Task Spec의 `constraints.negative` 필드에 원문 그대로 기록한다
- L3 아카이브 단계에서도 삭제하지 않는다
- 압축 시 이 필드만큼은 원문 보존을 최우선으로 한다

예시:
```json
"constraints": {
  "negative": [
    "lodash 사용 금지 (번들 크기 이슈)",
    "localStorage에 세션 토큰 저장 금지 (보안 정책)",
    "직접 DOM 조작 금지 (React 상태 관리 정책)"
  ]
}
```

## 다중 요청 처리 원칙

| 상황 | 처리 방법 |
|---|---|
| 각 요청의 독립성 | 독립된 task_id를 부여받아 개별 관리 |
| 선행 Task 결과가 후행 Task에 영향 | L2 요약을 후행 Task의 `relevant_history`로 명시적으로 전달 |
| 선행 Task 실패 | 실패 원인과 해결 여부를 후행 Task 컨텍스트에 포함 |
| 선행 Task FAILED + depends_on 존재 | 후행 Task를 즉시 **HOLD** 상태로 전환, 사용자에게 즉시 보고 |
| 전체 요청이 하나의 목표 공유 | Analyst가 상위 목표를 별도 메타 컨텍스트로 유지 |

## 에이전트별 컨텍스트 범위

| 구성 요소 | Analyst | Researcher | Generator | Validator |
|---|:---:|:---:|:---:|:---:|
| 전체 대화 이력 | ✅ | ❌ | ❌ | ❌ |
| Task Spec | ✅ | 일부 | ✅ | ✅ |
| 이전 작업 요약 (L2) | ✅ | 필요 시 | 필요 시 | ❌ |
| 규칙 문서 | ✅ | ❌ | ✅ | ✅ |
| Research Summary | ✅ | ❌ | 필요 시 | ❌ |
| Generator 결과물 | ✅ | ❌ | ❌ | ✅ |
| Validator 결과 | ✅ | ❌ | FAIL 시 | ❌ (Tier 3: A↔B 공유 금지) |
