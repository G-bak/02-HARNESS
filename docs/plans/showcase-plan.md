# LandingHub — 프로젝트 기획서 & 구현도

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | LandingHub |
| **슬로건** | 국내 최고의 랜딩페이지 쇼케이스 |
| **파일** | `showcase.html` (단일 파일, 의존성 없음) |
| **타겟** | 국내 프론트엔드 개발자 / 디자인 레퍼런스 탐색자 |
| **목적** | 국내 개발자가 만든 랜딩페이지를 엄선·전시하고, 제출·알림 수요를 확보 |

---

## 2. 기획 배경 & 목표

- 국내에 Dribbble·Landingfolio 같은 **한국어 랜딩페이지 아카이브**가 없음
- 개발자들이 레퍼런스를 찾기 위해 해외 사이트에 의존하는 문제를 해소
- **3가지 수익 모델** 선점: 광고(배너/사이드), 프리미엄 등재, 제출 큐레이션 수수료

### 핵심 목표

1. 방문자가 영감을 얻고 레퍼런스를 발견할 수 있는 **쇼케이스 플랫폼** 구축
2. 개발자가 자신의 작품을 **무료로 제출**하고 피드백을 받을 수 있는 채널 마련
3. 이메일 **뉴스레터 구독자 확보**로 재방문율 제고

## 3. 사이트맵 & 섹션 구성

```
LandingHub (showcase.html)
├── NAV            — 로고 / 메뉴 / 제출 버튼
├── HERO           — 헤드카피 / 검색창 / 통계 스트립
├── MARQUEE        — 기술 스택 무한 스크롤 띠
├── AD STRIP       — 상단 가로 광고 (728×90)
├── MAIN LAYOUT
│   ├── 필터 Pills  — 전체/SaaS/포트폴리오/커머스/스타트업/앱
│   ├── 정렬/뷰 토글 — 최신순·인기순 / 그리드·리스트
│   ├── 카드 그리드   — .lp-card (썸네일 + 메타)
│   └── SIDEBAR
│       ├── 카테고리 목록
│       ├── 사이드 광고 (250×250)
│       ├── 작품 제출 CTA
│       └── 인기 태그 클라우드
├── NOTIFY         — 이메일 알림 구독
├── HOW IT WORKS   — 3단계 프로세스 설명
├── BOTTOM AD      — 하단 가로 광고 (728×90)
└── FOOTER         — 링크 / 문의 / 저작권
```

## 4. 기능 명세

### 4-1. 카드 시스템

| 항목 | 명세 |
|------|------|
| 데이터 소스 | JS 내부 `CARDS` 배열 (정적) |
| 카드 필드 | `id`, `title`, `author`, `tag`, `tags[]`, `url`, `thumb`, `date`, `views`, `likes` |
| 썸네일 | 16:10 비율, hover 시 1.06배 스케일 |
| 오버레이 | hover 시 "사이트 방문 →" 버튼 노출 |
| 좋아요 | 토글 방식 (♡ / ♥), 로컬 상태만 유지 |
| 카테고리 배지 | 썸네일 좌상단에 절대 위치 |

### 4-2. 필터링 & 검색

| 항목 | 명세 |
|------|------|
| 카테고리 필터 | 상단 Pills + 사이드바 카테고리 동기화 |
| 텍스트 검색 | 제목·작성자·태그 실시간 검색 (소문자 정규화) |
| 빈 결과 | emptyState 표시 (현재 주석 처리) |
| 정렬 | 최신순 / 인기순 (UI만 구현, 실제 정렬 로직 미완) |
| 뷰 전환 | 그리드 (auto-fill) / 단일 열 리스트 |

### 4-3. 스켈레톤 로딩

- 카드 데이터가 없을 때 9개 shimmer 스켈레톤 표시
- 카드 추가 시 자동으로 숨겨짐

### 4-4. 알림 구독 (NOTIFY)

- 이메일 입력 + "알림 받기" 버튼
- Cloudflare Worker + D1 백엔드 연동 완료
- 세부 구현 기준은 [newsletter-admin-plan.md](./newsletter-admin-plan.md)를 우선 참조

### 4-5. UX / 인터랙션

| 기능 | 구현 방식 |
|------|----------|
| 스크롤 reveal | `IntersectionObserver` + `.rv` / `.show` 클래스 |
| 버튼 글로우 | mousemove → CSS 변수 `--x`, `--y` radial-gradient |
| ⌘K 단축키 | 검색창 포커스 |
| 마퀴 띠 | `<template>` 복제 × 2, CSS `translateX(-50%)` |

- 폰트: `Inter` (Google Fonts, 300–900)

- 그라디언트 텍스트: `.g-text` (퍼플 → 시안 → 핑크, 6s 루프)

### 애니메이션

| 클래스/ID | 효과 |
|-----------|------|
| `animate-float1~3` | 배경 오브 부유 (14s/18s/11s) |
| `animate-marquee` | 기술 태그 수평 스크롤 (28s) |
| `animate-shimmer` | 스켈레톤 글리터 (1.8s) |
